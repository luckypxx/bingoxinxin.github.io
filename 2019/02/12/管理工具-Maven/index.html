管理工具---Maven

  • JavaEE

Maven用于管理各种第三方jar和打包发布并制定标准化项目结构

标准化项目结构

在顶层有一个名字为pom.xml的配置文件

在顶层还有一个目录叫做src

在src目录中有两个文件夹，一个是main文件夹用于存放源代码，一个是test文件夹用于存放测试代码

Maven项目的构建

  • 通过命令行构建
  • 使用开发工具IDEA

Maven中的三个关键配置

  • groupId：项目所属的公司或组织名称-用于后续包名称定义

  • artifactId:项目名称

  • version：项目版本号

pom.xml中的关键配置

Maven中添加依赖

百度搜索maven，点开maven依赖，查找相关jar包，确定其版本号。将其关键配置信息复制到文件中

Maven生命周期

  • compile
  • clean清空target目录
  • test执行测试用例
  • package将项目打包

在执行package将项目打包过程中会默认执行test命令运行所有测试用例，只有当所有测试用例均通过项目才会打包成功

打包过程中不再执行测试用例代码直接将项目打包

打包生成的结果是一个jar包或是war包（web），命名按照artifactId-version来命名

  • 构建可执行jar

1、在pom.xml添加一个打包插件并指明主类来使得打包生成的jar具有可执行能力

2、使用mvn package将项目打包

3、java -jar打包后的jar包

  • install安装构建到本地仓库

mvn install

本机中的其他项目可以使用dependency来引入项目，maven寻找的jar包顺序：

先从本地仓库中找，若不存在再去中央仓库中寻找，若中央仓库中都没有则报错

  • 匹配jar包的规则：根据pom文件中指定的dependency中的三个关键配置（groupId-artifactId-version）

  • deploy安装构建到中央仓库

  • site生成网站

mvn site

将项目信息、依赖等生成网站

Maven使用掌握点

  • 重点掌握IDEA创建Maven项目
  • 使用pom文件添加依赖
  • 会使用插件构建可执行jar包
