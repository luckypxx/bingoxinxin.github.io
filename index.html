Java虚拟机中方法调用的执行

虚拟机中方法重载的实现

  • 方法重载的概念

在同一个类中定义名字相同，参数类型必须不同的方法。

  • 编译器选取重载方法的过程

1.不考虑对基本数据类型自动拆装箱及可变参数

2.允许自动拆装箱，但不允许可变参数列表

3.允许自动拆装箱和可变参数列表

若Java编译器在同一阶段中找到了多个适配的方法，则其会根据形式参数类型的继承关系进行选择（即优先选择子类方法）

JVM的静态绑定和动态绑定

所谓静态绑定和动态绑定其实对于的就是动态多态和静态多态

  • Java虚拟机识别方法的方式

Java虚拟机识别方法的关键在于类名、方法名、以及方法描述符（参数类型和返回值类型）

  • Java虚拟机中重写的判定

如果子类定义了与父类中非私有、非静态方法同名的方法，两个方法的参数类型以及返回值类型一致

调用指令的符号引用

在编译过程中，Java编译器会暂时用符合引用来表示该目标方法，符号引用储存在class文件的常量池之中

对于非接口符号引用，假定该符号引用所指向的类为 C，则 Java 虚拟机会按照如下步骤进行查找。

  1. 在 C 中查找符合名字及描述符的方法。
  2. 如果没有找到，在 C 的父类中继续搜索，直至 Object 类。
  3. 如果没有找到，在 C 所直接实现或间接实现的接口中搜索，这一步搜索得到的目标方法必须是非私有、非静态的。并且，如果目标方法在间接实现的接口中，则需满足 C 与该接口之间没有其他符合条件的目标方法。如果有多个符合条件的目标方法，则任意返回其中一个。

  • HashMap的无参构造

  • HashMap的put方法

调用put()实际上调用的是putVal()，那接下来就看一下putVal()方法,后面的hash()方法是用来求哈希值的，我们稍后再讲

  • putVal()
  • hash()

hash()实际上是对hashCode()进行一层包装

return语句：若key为null则返回hash值0，否则需调用hashCode()求hash值。

求出key的hashCode值，将其与h右移16位进行异或。

注意：>>>是无符号的右移，>>是有符号的右移

问题1：

那么我们不得不思考一个问题，求hash值直接调用hashCode()不就可以了吗，为什么还要进行一层包装呢？

先来看一下hashCode()方式是如何定义的

我们可以看到hashCode()的返回值是int类型，int大概是2^31左右，如果hash桶的数量这么大，基本上不会出现哈希碰撞，那么也就不会对哈希桶进行扩容，那么HashMap使用的哈希表和数组也就没什么区别了。

问题2：

为何取出key值得高16位右移参与hash运算？

因为hash基本上是在高16位进行hash运算

问题3：

为何hashMap的容量为2^n

(n -1)&hash :当n为2^n,此时的位运算就相当于hash%（n-1）

Java中的深浅拷贝

深浅拷贝

拷贝的执行函数

并不是所有的类都可以被拷贝，需实现Cloneable接口，并且在类中自定义clone()调用Object类提供的继承权限clone()

若不实现Cloneable接口，在调用clone()时，会抛出CloneNotSupportedException异常。

浅拷贝

对于浅拷贝而言，拷贝出来的对象仍然保留原对象的所有引用。

只要拷贝对象和原对象中引用的属性发生改变，其所有的该引用属性都会发生改变

深拷贝

对于深拷贝而言，拷贝出来的对象会产生新的对象。

对于拷贝的原对象和拷贝出来的对象来说，改变一个引用的属性，互不影响

对象中包含的引用只是自定义引用
